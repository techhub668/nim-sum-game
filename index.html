Nim Sum Theory

What is the Game of Nim?

Nim is a mathematical strategy game where two players take turns removing objects from distinct piles. On each turn, a player must remove at least one object, and may remove any number of objects provided they all come from the same pile. The goal is typically to be the player who takes the last object.

Understanding Binary Numbers

Before we learn the winning strategy, we need to understand binary numbers - the language computers use!

What is Binary?

In our everyday life, we use the decimal system (base 10) with digits 0-9. Binary is the base 2 system that only uses two digits: 0 and 1.

Each position in binary represents a power of 2, just like decimal positions represent powers of 10.

The 4-2-1 Method (Easy Way to Understand Binary!)

Think of binary as breaking down numbers into groups of powers of 2:

Position ... 16 8 4 2 1
Power of 2 ... 24 23 22 21 20

To convert a number to binary, ask: "Can I fit this power of 2 into my number?"

  • If YES, write 1 and subtract that value
  • If NO, write 0 and move to the next smaller power

Example: Why is 3 = 011 in binary?

Let's break down 3 using powers of 2:

  • Can we fit 4 into 3? NO (4 > 3) → write 0
  • Can we fit 2 into 3? YES (2 ≤ 3) → write 1, remaining: 3-2 = 1
  • Can we fit 1 into 1? YES (1 ≤ 1) → write 1, remaining: 1-1 = 0

Result: 3 = 0×4 + 1×2 + 1×1 = 011

Example: Why is 5 = 101 in binary?

Let's break down 5:

  • Can we fit 4 into 5? YES → write 1, remaining: 5-4 = 1
  • Can we fit 2 into 1? NO (2 > 1) → write 0
  • Can we fit 1 into 1? YES → write 1, remaining: 0

Result: 5 = 1×4 + 0×2 + 1×1 = 101

Example: Why is 7 = 111 in binary?

Let's break down 7:

  • Can we fit 4 into 7? YES → write 1, remaining: 7-4 = 3
  • Can we fit 2 into 3? YES → write 1, remaining: 3-2 = 1
  • Can we fit 1 into 1? YES → write 1, remaining: 0

Result: 7 = 1×4 + 1×2 + 1×1 = 111

What is XOR (Exclusive OR)?

XOR is a special operation that compares two bits (0 or 1) and follows this simple rule:

"If the bits are DIFFERENT, result is 1. If they're the SAME, result is 0."

XOR Truth Table:

ABA XOR BWhy?
000Same → 0
011Different → 1
101Different → 1
110Same → 0

Think of XOR as "Odd One Out"

When you XOR multiple numbers together column by column:

  • If there's an ODD number of 1s in a column → result is 1
  • If there's an EVEN number of 1s in a column → result is 0

This is why pairs of identical bits "cancel out" to zero!

The Nim Sum (XOR Operation)

The key to winning Nim is the Nim Sum, calculated by XORing all pile sizes together.

Example: Piles of 3, 5, 7

3 in binary:  011  (0×4 + 1×2 + 1×1)
5 in binary:  101  (1×4 + 0×2 + 1×1)
7 in binary:  111  (1×4 + 1×2 + 1×1)
              ---
XOR result:   001 = 1 (Nim Sum)

Column by column:
- 4's column: 0 XOR 1 XOR 1 = 0 (even 1s)
- 2's column: 1 XOR 0 XOR 1 = 0 (even 1s)
- 1's column: 1 XOR 1 XOR 1 = 1 (odd 1s)

Since Nim Sum ≠ 0, the first player can guarantee a win!

The Easy 4-2-1 Pairing Method

Here's a simpler way to think about Nim Sum without doing binary math in your head!

The Pairing Strategy

For each power of 2 (1, 2, 4, 8, 16...), count how many piles have that component:

  • If the count is EVEN (0, 2, 4...) → That position is "balanced" (good!)
  • If the count is ODD (1, 3, 5...) → That position is "unbalanced" (contributes to Nim Sum)

Goal: Make ALL positions balanced (even counts) to achieve Nim Sum = 0!

Example: Piles of 3, 5, 7 using Pairing

Pile Has 4? Has 2? Has 1?
3 = 2+1 No Yes Yes
5 = 4+1 Yes No Yes
7 = 4+2+1 Yes Yes Yes
Count: 2 (even) 2 (even) 3 (odd!)

The "1" column has an odd count (3), so Nim Sum ≠ 0.

Winning move: Remove 1 from pile of 3, making it 2. Now all columns are even!

Example: Piles of 1, 3, 5, 7 - A Famous Nim Position!

Pile Has 4? Has 2? Has 1?
1 = 1 No No Yes
3 = 2+1 No Yes Yes
5 = 4+1 Yes No Yes
7 = 4+2+1 Yes Yes Yes
Count: 2 (even) 2 (even) 4 (even)

All columns are even! Nim Sum = 0. The first player is in a LOSING position!

This is a famous "losing" starting position in Nim.

The Winning Strategy

1
Calculate the Nim Sum

XOR all pile sizes together (or use the pairing method).

2
If Nim Sum ≠ 0: You're in a WINNING position!

Make a move that results in Nim Sum = 0 for your opponent.

3
If Nim Sum = 0: You're in a LOSING position

Any move you make will give your opponent Nim Sum ≠ 0.

Finding the Winning Move

When Nim Sum ≠ 0, to find which pile to reduce:

  1. Calculate the Nim Sum (N) of all piles
  2. Find a pile P where P XOR N < P
  3. Reduce that pile to (P XOR N)

Example: Piles of 22 and 36

22 in binary: 010110  (16+4+2 = 22)
36 in binary: 100100  (32+4 = 36)
              ------
Nim Sum:      110010 = 50

Check pile 22: 22 XOR 50 = 36 (NOT less than 22, can't use)
Check pile 36: 36 XOR 50 = 22 (22 < 36, YES!)

Solution: Take 14 from pile of 36 (36 - 22 = 14)
Result: Both piles at 22, Nim Sum = 0

Using Pairing Method for 22 and 36:

Pile 32 16 8 4 2 1
22 = 16+4+2 No Yes No Yes Yes No
36 = 32+4 Yes No No Yes No No
Count: 1 (odd) 1 (odd) 0 (even) 2 (even) 1 (odd) 0 (even)

Columns 32, 16, and 2 are odd. We need to balance them!

Solution: Change pile 36 to 22. New 22 = 16+4+2, so now both piles are identical and all columns are even (paired)!

Single Pile Strategy (Subtraction Game)

When you can only take 1 to k objects per turn from a single pile:

  • Winning positions: Pile size NOT divisible by (k+1)
  • Losing positions: Pile size divisible by (k+1)

Strategy: Leave your opponent with a multiple of (k+1) after your turn.

Example: 12 matchsticks, max take 3

k+1 = 4, Multiples: 0, 4, 8, 12...

Since 12 is a multiple of 4, the first player is in a LOSING position (if opponent plays perfectly).

Why Does This Work?

The XOR/pairing strategy works because:

  • The end state (all piles empty) has Nim Sum = 0
  • From any position with Nim Sum ≠ 0, there exists a move to make Nim Sum = 0
  • From any position with Nim Sum = 0, ALL moves result in Nim Sum ≠ 0

This creates a "trap" - once you achieve Nim Sum = 0, your opponent cannot escape!

The Key Insight

When all columns are "paired" (even counts), removing ANY amount from ONE pile will break at least one pairing. Your opponent will always give you an unbalanced position, and you can always rebalance it!
